# PyPI Publishing Setup Guide

Your workflow is ready, but you need to authenticate with PyPI. Choose ONE approach:

## Option 1: API Token Authentication (Simpler, Recommended for Getting Started)

### Step 1: Generate PyPI API Token

1. Go to https://pypi.org/manage/account/tokens/
2. Click **"Add API token"**
3. Name: `arceion-zorion-github-actions`
4. **Copy the token immediately** (you can only see it once)
5. Format: `pypi-AgEIcHlwaS5vcmc...`

### Step 2: Add Token to GitHub Secrets

1. Go to your GitHub repository
2. **Settings** → **Secrets and variables** → **Actions**
3. Click **"New repository secret"**
4. **Name:** `PYPI_API_TOKEN`
5. **Value:** Paste your PyPI token from Step 1
6. Click **"Add secret"**

### Step 3: Verify Workflow Config

Your workflow is already configured to use the token:
```yaml
- name: Publish to PyPI
  uses: pypa/gh-action-pypi-publish@release/v1
  with:
    password: ${{ secrets.PYPI_API_TOKEN }}
```

The `password` field will use your API token for authentication.

### Step 4: Release

1. Update version in `pyproject.toml`
2. Commit and push
3. Create a GitHub Release with tag `v0.2.0`
4. Workflow automatically publishes ✅

---

## Option 2: Trusted Publishers (More Secure, Recommended Long-term)

Trusted publishers allow PyPI to verify that the release comes from YOUR GitHub repository without storing any tokens.

### Prerequisites

- Your GitHub repository must be public (trusted publishers only work with public repos)
- PyPI account with publish permissions to `arceion-zorion`

### Step 1: Add Trusted Publisher on PyPI

1. Go to https://pypi.org/manage/project/arceion-zorion/
2. Click **"Publishing"** tab
3. Click **"Add a publisher"**
4. Select **"GitHub"**
5. Fill in:
   - **Repository name:** `Arceion/arceion-zorion`
   - **Workflow name:** `publish.yml`
   - **Environment name:** (leave empty for releases, or set to specific environment)
6. Click **"Save"**

### Step 2: Update Workflow

Update `.github/workflows/publish.yml` to remove the `password` field:

```yaml
- name: Publish to PyPI
  if: github.event_name == 'release' || (github.event_name == 'workflow_dispatch' && github.event.inputs.target == 'pypi')
  uses: pypa/gh-action-pypi-publish@release/v1
  # NO password field - uses OIDC instead
```

The workflow will now use OIDC tokens instead of API tokens ✅

### Step 3: Release

1. Update version in `pyproject.toml`
2. Commit and push
3. Create a GitHub Release with tag `v0.2.0`
4. Workflow automatically publishes using OIDC ✅

## Troubleshooting

### "invalid-publisher: valid token, but no corresponding publisher"

**You're in Option 2 mode but haven't registered a trusted publisher.**

**Solution:**
- Either use Option 1 (add API token to GitHub secrets)
- Or complete Option 2 setup (register trusted publisher on PyPI)

### "Invalid or missing token"

**Your API token is incorrect or expired.**

**Solution:**
1. Go to https://pypi.org/manage/account/tokens/
2. Revoke the old token
3. Generate a new one
4. Update the GitHub secret

### "Unauthorized: Invalid authentication"

**Token has wrong permissions or is for wrong package.**

**Solution:**
1. Check token is for the right package: `arceion-zorion`
2. Check token has "Entire account" or "Project: arceion-zorion" scope
3. Re-generate if needed
